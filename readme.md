# 脑机接口智能家居系统项目说明

## 1. 项目概述
围绕“脑机接口驱动智能家居”这一主题，整合了 SSVEP 视觉刺激、运动想象（MI）、情感识别、语音控制以及 Brain pods 扩展能力，构建出一套从“脑信号输入”到“家居响应反馈”，再到“健康状态感知与神经调节”的完整演示方案。

## 2. 系统整体功能
系统围绕“控制、感知、调节”三条主线展开，形成以下几类核心能力：

- 脑机控制输入：通过 SSVEP 和 MI 实现注视选择与意念触发。
- 状态感知展示：通过情感识别、频段分析和生命体征联动展示用户状态。
- 自然语言控制：通过语音识别与文本回退实现家居指令输入。
- 健康管理延伸：通过 Brain pods 子系统展示疲劳检测、健康评估和助眠/专注调节。

这些功能不是简单拼接，而是围绕“人机交互闭环”统一组织：用户输入意图，系统做出响应，同时反馈状态变化和健康提示。

## 3. 主要模块功能介绍

### 3.1 SSVEP 视觉刺激模块
SSVEP 模块用于模拟基于视觉诱发电位的脑机控制方式，是系统中最典型的“注视式选择”入口。页面通过多个闪烁目标构成可交互刺激区，用户可通过观察目标完成意图表达。

模块功能包括：

- 多个独立闪烁目标，对应不同家居动作。
- 可调节刺激频率与对比度，支持演示参数变化。
- 开始、停止、全屏等控制，便于展示刺激状态。
- 快捷键和点击双重触发方式，提高交互演示效率。
- 状态面板同步显示当前刺激与触发记录。

这一模块的重点是体现“可配置、可切换、可观察”的脑机刺激演示能力。

### 3.2 MI 运动想象模块
MI 模块以按摩椅场景为载体，将运动想象任务映射为具体部位的控制操作，强调“单任务互斥”和“状态持续窗口”的设计。

模块功能包括：

- 提供大腿、手臂、肩颈等多个想象任务入口。
- 同一时刻只允许一个部位进入激活状态，避免冲突。
- 激活后自动进入计时窗口，到期后恢复待机。
- EEG 波形在激活时增强，在待机时趋于平缓。
- 状态灯、按钮高亮和文字提示联动变化。

该模块的核心价值在于，它不仅展示“点击后变化”的简单效果，还模拟了脑机控制中常见的任务状态管理机制。

### 3.3 情感与生命体征模块
情感模块负责展示用户脑电状态与生理状态的联合分析，是系统中从“控制”转向“感知”的关键部分。页面通过多通道 EEG、频段统计和情感分类结果，构建出可解释的状态视图。

模块功能包括：

- 12 通道 EEG 模拟波形，增强整体脑电感知效果。
- Alpha、Beta、Theta、Delta 等频段数据动态刷新。
- 根据频段特征自动判定情感状态，如放松、中性、焦虑、压力等。
- 同步展示 SpO2、血压、心率等生命体征卡片。
- 给出状态说明、医学解释和风险提示，提升可读性。

这个模块的设计目标是将“单看脑波”升级为“脑电 + 生理指标”的综合视角，让系统更像一个状态监测平台，而不仅是一个控制面板。

### 3.4 语音智能管家模块
语音模块模拟智能家居中的自然语言控制入口，用于补充脑机输入之外的交互方式。它支持语音识别、指令解析和设备状态同步，是系统里最贴近日常使用场景的部分。

模块功能包括：

- Web Speech API 语音识别，支持实时结果显示。
- 文本输入回退，在麦克风不可用时仍能操作。
- 中英文关键词识别，兼顾多语言演示。
- 灯光、窗帘、空调、音乐、场景模式等常见家居控制。
- 温度数字解析和增减操作，适合演示精细控制逻辑。
- 对话记录与设备状态卡片同步更新。

这一模块体现的是“自然语言作为统一控制入口”的思路，能够把脑机交互之外的传统语音控制整合进同一个智能家居系统中。

### 3.5 Brain pods 扩展子系统
Brain pods 是系统的延伸部分，重点不再只是“控制设备”，而是把项目拓展到“健康管理”和“神经调节”的方向。它让整个项目从一个交互展示 Demo，进一步变成一个更完整的脑健康应用原型。

#### 3.5.1 疲劳检测
疲劳检测子模块用于模拟左右脑脑电状态和时间感知下的疲劳判断。

功能包括：

- 左右脑双通道 EEG 波形展示。
- 疲劳指数计算与等级划分。
- 结合早晨、中午、夜间等时段给出不同提示。
- 显示医生建议和补充说明。
- 频段活跃度表帮助用户理解状态来源。

#### 3.5.2 健康检测
健康检测子模块用于展示一段时间内的趋势性健康状态，强调长期观察和风险评估。

功能包括：

- 自动生成近 7 天多项指标数据。
- 使用表格展示情感、活跃度、睡眠、稳定度等维度。
- 通过雷达图对比用户状态与参考水平。
- 输出风险等级和主要风险原因。

这个模块更偏向“周报式健康管理”，适合展示持续监测的概念。

#### 3.5.3 助眠/专注调节
助眠/专注子模块用于模拟个性化神经调节策略，是 Brain pods 中最具未来感的部分。

功能包括：

- 在助眠和专注两种模式之间切换。
- 提供低、中、高三档强度策略。
- 根据模式与强度同步更新方案、提示和效果说明。
- 高强度时弹出二次确认，强调使用安全性。
- 展示使用前后指标变化，增强效果对比感。

该模块的重点是“干预建议 + 风险提示 + 效果反馈”的组合表达。

## 4. 技术实现特点
项目采用纯前端技术实现，整体结构轻量，适合本地打开和展示。

- 使用 HTML5 和 CSS3 构建页面与样式。
- 使用原生 JavaScript 实现交互逻辑和状态切换。
- 使用 Canvas 2D 生成波形、趋势图和雷达图。
- 使用 Web Speech API 支持语音识别。
- 使用 `requestAnimationFrame` 处理高频动画和实时刷新。

在实现方式上，项目强调“模块独立、状态清晰、展示直观”，方便后续替换为真实数据源或接入外部设备。

## 5. 运行方式
建议使用本地静态服务器打开项目，以保证语音识别、资源加载和页面跳转更稳定：

```bash
python -m http.server 8000
```

然后访问：

```text
http://localhost:8000
```

也可以直接打开 `index.html`，但部分浏览器下语音能力和跨页面资源加载可能不如本地服务稳定。

## 6. 使用说明与注意事项

- 所有 EEG、生理指标和健康评估数据均为模拟结果，不可用于医疗诊断。
- 情感判断、疲劳评估和风险等级属于规则演示逻辑。
- SSVEP 页面包含高频闪烁元素，请避免长时间连续注视。
- 语音功能依赖浏览器实现和麦克风权限，不同设备体验会有差异。

## 7. 项目总结
这个系统的核心价值在于把“脑机控制”“状态感知”“语音交互”和“健康管理”放在同一套原型里统一演示，形成一个从输入、响应到调节的完整逻辑链。它既能展示未来感，也保留了清晰的模块边界，适合作为课程作业、竞赛项目或概念验证原型。
